Separation of Concerns


Separation of concerns (SoC) is the design concept in which each concern in the code is given its distinct section in the code.


In coding, a concern is a distinct behavior that a unit of code in a script or program has to perform. Therefore, each concern in the program must be in a separate section, typically in a separate method. Each method in a script should perform its own independent task. Methods should have one and only one job.

The design principle of SoC is important in coding as it allows us to have a clear code that is divided into small, manageable pieces. This way, it easier to work with each small unit of code as you don't need to know much about it besides the one task it is designed to accomplish.

An example of the use of SoC in our calculator program is in calculations.py:


Here we can see how each method only accomplishes one task or concern. We can see that there is an add_calculation() method that only adds a calculation to history, a clear_history() method that only clears the history, a count_calculations() method that only returns the number of calculations in history, and etc.

These methods only accomplish the one job they are assigned to do and nothing else and therefore are broken into small, manageable pieces of code. Each of the methods are indepedent and are used together in order to build up our calculator program.

Overall, applying SoC allows us to have short segments of code with clear intent, making them easier to write, understand, use, and fix.
