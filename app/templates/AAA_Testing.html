AAA Testing

AAA is a pattern of testing that helps determine whether the functions in a program are working as expected. Testing is important because it helps us pinpoint problem areas in our code in development before deploying problematic code. It also allows us to keep track of where code breaks as APIs change and new code is added later on.
To start, AAA stands for Arrange, Act, and Assert.

Our calculator program includes tests for each function based on the AAA testing pattern discussed below.


For this article, we will be using the following function, add_numbers(), found in calculator.py for testing using the AAA testing pattern:


ARRANGE

Arranging is the process of arranging the inputs and targets needed to set up for the test cases. In order to test the functionality of a basic unit of code, we need to make sure we have the necessary operations, settings and/or variables set up and prepped.

In our example function from the calculator class, the arranging step should look like this:

my_tuple = (1, 1)

Here, we are arranging a tuple for the add_numbers() method as it takes in a tuple of numbers to add. We need to have a tuple of numbers to add to make sure we can test the functionality.


ACT

The Act stage is where we call the function to be tested. This is typically a function or method call as in this case, we are testing a method. The action is called so we can target whether it is behaving correctly or not in the next step.

In our example, we would be calling the add_numbers() function that we are testing like this:

Calculator.add_numbers(my_tuple)


ASSERT

Lastly, the Assert stage is where we write an assert statement to make sure we get our expected outcome as a result of our function call. The assert function verifies that we received the correct response for the behavior and it lets us determine if the test passes or fails.

In our example, we would call the following assert statement to make sure that the last calculation result value is 2 as is expected of adding the numbers we fed the function in the arrange stage:

assert Calculator.get_last_result_value() == 2


If we run this test (using pylint) we should be able to see if our function passes the test and whether or not it functions as we expect it to. This helps us narrow down issues in our code and makes sure that every unit of code in our program functions properly. Each function in a program should have a test function starting with the name starting with 'test_' in order for us to run pytest and see if our code works as expected.

You can learn more about pytest here: https://docs.pytest.org/en/6.2.x/contents.html.

AAA testing helps us do basic unit testing and feature testing following the behavior-driven development concept of Given-When-Then.
