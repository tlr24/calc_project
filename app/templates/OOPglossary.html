OOP Glossary


OOP or object-oriented programming uses many concepts including: Factory, instantiation, object, class, namespace, constructor, fixture, type hint, type cast, unit test, static, class method, facade, method, property, linting, etc.



pylint - Pylint is a tool used to catch errors in python code development as well as to make sure that python code reaches the PEP8 python standards of how and in what style python code should be written.

More information about how to install and implement pylint for your development environment can be found here: https://pylint.org/.

More information on the PEP8 standards and style guide can be found here: https://www.python.org/dev/peps/pep-0008/.

factory - A factory makes objects instead of just instantiating them. This helps centralize object creation.

An example of a factory method in our calculator program is in the calculation class on line 12 where we make a create function that always takes the first parameter of the class to instantiate as an object. It also takes in the tuple used in the __init__ function (line 7) to instantiate the object with the values established in the constructor method.
Note: The full calculation class can be viewed here: https://github.com/tlr24/calc_project/blob/main/calculator/operations/calculation.py.

instantiation - Instantiation is creating an instance of a class. When an object is instantiated, it inherits all of the data and methods from the class.

object - An object is an instance of a class. It is a copy of the class as the class is the blueprint while an object actually carries the properties and methods.

class - A class is a blueprint or an outline used to instantiate objects. It defines the data and functions to be instantiated with the object.
A class in python can be defined using the following syntax: class SampleClass:

namespace - A namespace is a collection of names of objects including properties and methods. Namespaces help define the scope of the names within a class.

For example, we can have different get_result() methods in both the addition.py and subtraction.py classes that can be used within the scope of their namespaces or scripts using the same name for different purposes.

constructor - A constructor is a method that gets called to instantiate an object. In python, the constructor method is created with the __init__ function.

An example of an __init__ constructor method can be found in our calculation.py class. The constructor method can be called to instantiate a calculation object by calling the following line: Calculation()
Note: The full calculation class can be viewed here: https://github.com/tlr24/calc_project/blob/main/calculator/operations/calculation.py.

unit test - A unit test is a testing method that tests small, individual units of code to make sure they function properly as expected. Unit tests help isolate where code may be broken to identify issues and fix programs quickly and efficiently.

static method - Static methods are utility functions that are there to do actions and calculations. They do not take up memory. They help for usability.

An example of a static method in our calculator program is the clear_history() method in calculations.py.


The clear history method is utility that is only used to clear the calculations history data. It does not need to store any of the data in the class, it just needs to perform a static function every time it is called.

class method - A class method is a method that is bound to the class. It does not rely on a class instance, like a static method, however, class methods do take a parameter of class (the first parameter) that refers to the overall class itself. Unlike a static method, it deals with the class and its properties.

An example of a class method in our calculator program is the factory method in calculation.py. In python, class methods must include the @classmethod decorator above the function signature.


facade - A facade is a design pattern used to hide the complexities of a program by providing an interface for the client to use. An example of a facade would be the calculator class. The calculator class imports from the Calculations class which imports from the operation classes. This way, the calculator because a simple interface for users to use as it hides all the complexities found in calculation.py, addition.py, subtraction.py, multiplication.py, calculations.py, etc.



method - A method is a function or a block of code that runs when called. Methods are useful in object-oriented programming as they allow for code reusability. Methods in python can be defined using the def keyword.

The calculator program uses methods in almost every python script. Methods are defined using the following format called a method signature: def example_function(parameters):

The method signature includes the defined method name and parameters.

Note: The full calculator program can be viewed here: https://github.com/tlr24/calc_project.

property - A property is an interface to an attribute of an object or class. Properties hold data of an object. Property methods allow us to get, set, and delete attributes of an object defined in a class, making it easier to work with the data within its scope.
